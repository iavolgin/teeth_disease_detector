# Teeth Disease Detector

Проект для автоматической обработки стоматологических изображений. Приложение принимает путь к папке с изображениями, для каждого изображения выполняет детекцию зубов, вычисляет метаданные и классифицирует найденные зубы, после чего сохраняет размеченные результаты в папку `results` внутри выбранной директории.

## Как работает пайплайн

1. `pkg/teeth_detector.py` находит зубы на изображении и возвращает bounding boxes.
2. `pkg/metadata.py` вычисляет метаданные для каждого найденного зуба.
3. `pkg/teeth_classifier.py` загружает обученную модель и предсказывает номер зуба.
4. `usecase/pipe.py` связывает все этапы в единый пайплайн.
5. `cmd/main.py` запускает приложение через консольный интерфейс.

## Локальный запуск

Запуск из корня проекта:

```bash
python cmd/main.py
```

После запуска приложение выведет инструкцию в консоль и попросит указать путь к папке с изображениями.

Также можно передать путь сразу:

```bash
python cmd/main.py C:\path\to\images
```

Что произойдет после запуска:

1. Приложение найдет все поддерживаемые изображения в указанной папке.
2. В этой же папке будет создан каталог `results`.
3. В `results` будут сохранены итоговые изображения с нанесенными bounding boxes и номерами зубов.

Поддерживаемые форматы изображений:

- `.jpg`
- `.jpeg`
- `.png`
- `.bmp`
- `.tif`
- `.tiff`

## Что проверяет GitHub Actions

Workflow `.github/workflows/app-smoke.yml` запускается на `push` и `pull_request` и проверяет:

1. Что ключевые Python-файлы проекта проходят проверку синтаксиса через `py_compile`.
2. Что CLI-приложение корректно запрашивает путь к папке в интерактивном режиме.
3. Что пайплайн в smoke-тесте создает папку `results` и сохраняет итоговое изображение без JSON-артефактов.

Важно: GitHub Actions сейчас выполняет именно smoke-проверку приложения, а не полный инференс реальной модели.
